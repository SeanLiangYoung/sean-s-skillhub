# 本地环境如何与团队约定对齐（不必统一 IDE）

**结论：不需要统一 IDE。** 需要统一的是：**同一套仓库内可版本化的约定**、**与 CI 一致的运行时版本**、以及**合并前同一组命令**（见 [git-and-review.md](./git-and-review.md)）。各人可用 VS Code、Cursor、JetBrains、Neovim 等，只要能在终端执行相同脚本即可。

## 1. 「固化」到底固什么


| 层级           | 作用                               | 是否随 Git 版本管理                       |
| ------------ | -------------------------------- | ---------------------------------- |
| 文档与索引        | 入口文档、`docs/`、本目录说明               | 是                                  |
| 契约与检查        | 实现中的 schema、仓库提供的 drift/schema 脚本 | 是                                  |
| CI 定义        | 本仓库 `.github/workflows/`（或等价路径）   | 是                                  |
| 编辑器提示（可选）    | `.cursor/rules/`（若使用 Cursor）       | 是；**仅在使用 Cursor 且打开本仓库时生效**        |
| 本机解释器 / Node | 具体小版本                            | **否**，由每人安装；应用 **与 CI 对齐的版本号** |


**IDE 差异**只影响补全与 AI 集成方式；**门禁与契约**以仓库 + 终端命令为准。

## 2. 每人本地应做到的对齐点

1. **语言版本与 CI 一致**  
   以本仓库 CI 工作流或 `README` 声明为准；本地用 `pyenv`、`mise`、`asdf` 或官方安装器固定同主版本，避免「本地能跑、CI 失败」。
2. **依赖按仓库声明安装**  
   按各子项目 `requirements.txt`、`package.json` 或文档安装，不把个人全局包当事实源。
3. **环境变量不提交密钥、结构一致**  
   按仓库 `.env.example` 或文档中的变量表自建本地环境，勿将真实密钥提交到 Git。
4. **合并前跑与 CI 同逻辑的自检**  
   命令见 [git-and-review.md](./git-and-review.md) 与本仓库 CI 配置。

## 3. 可选增强（团队可逐步引入）


| 手段                                   | 说明                                                   |
| ------------------------------------ | ---------------------------------------------------- |
| **Dev Containers**（`.devcontainer/`） | 容器内固定运行时，最接近「人人同款」；维护成本略高。                      |
| **pre-commit**                       | 在 `git commit` 前自动跑 lint/format；需团队同意。           |
| **Makefile / `npm run check`**       | 把自检封成一条命令，减少抄错。                                        |
| **EditorConfig**                     | 统一换行、缩进风格，与 IDE 无关。                                  |


以上均为**可选**；最小集仍是：**文档 + 版本对齐 + 终端跑通自检**。

## 4. Cursor 与「非 Cursor」同事

- **`.cursor/rules/`** 只对 Cursor 用户在本仓库内生效，用于提醒「先读文档、勿臆造字段」等，**不能**替代实现代码与 `docs/`。
- 不使用 Cursor 的同事：**同一套流程**仍成立——阅读 [`engineering-principles.md`](./engineering-principles.md) 与入口文档，用终端执行自检。

## 5. 新人本地 checklist（建议）

- 克隆仓库，读完本目录 `README` 与仓库**入口文档**。
- 安装与 **CI 声明一致** 的语言运行时。
- 按仓库说明安装依赖并配置本地环境变量（若有模板则照模板）。
- 在仓库根执行 CI 对齐的自检命令，**全部通过**后再开始改契约类代码。
