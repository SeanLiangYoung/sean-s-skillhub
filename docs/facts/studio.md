# 本机运行契约（Skill Hub）

本文档描述 **Skill Hub 本地进程** 的固定行为，供排错、集成测试与 Agent 自动化时对齐预期。HTTP 路径清单见 [HTTP API 概览](../api.md)。

## 进程与网络

- **监听地址**：`127.0.0.1`（仅本机）。
- **起始端口**：环境变量 `PORT`，默认 `3456`。
- **端口占用**：若起始端口被占用，会依次尝试后续端口（最多 5 次，与实现中 `listenWithRetry` 一致）；以控制台输出的实际 URL 为准。

## 静态资源与 API-only 模式

- 生产启动时会在若干候选目录中查找**同时**包含 `index.html` 与 `assets/` 的构建产物（通常为仓库根目录下的 `dist/web/`）。
- 若未找到：进程仍以 **API-only** 模式运行，`GET /api/*` 可用；控制台会打印警告与已尝试路径。前端开发请使用 `npm run dev`（Vite 代理 `/api` 与 `/ws`）。

## 启动时行为

- 尝试清理回收站中的过期项（失败时静默忽略，不阻塞启动）。
- 执行一次全量技能扫描，便于在控制台展示统计；扫描失败时打印错误，服务仍监听。
- 若存在前端静态根且未设置 `SKILL_HUB_NO_OPEN=1`，会尝试用系统命令打开默认浏览器。

## 文件监听

- 默认启用 **chokidar** 监听（具体路径由扫描器推导）；变更经防抖后调用缓存失效并通过 WebSocket 广播。
- 设置 `SKILL_HUB_DISABLE_WATCH=1` 时不启动监听（适合 CI、`EMFILE` 受限环境）；列表仍可通过「一键扫描」或 `GET /api/scan` 刷新。

## WebSocket

- 路径：`/ws`。消息为 JSON 字符串；文件变更相关广播至少包含 `type: 'change'` 与事件信息（与实现一致即可）。
